AVIF — następna generacja formatów graficznych

AVIF (AV1 Image File Format) to nowoczesny format obrazków oparty na kodeku wideo AV1. Został stworzony przez Alliance for Open Media (Google, Mozilla, Apple, Microsoft) i oferuje:

  • Do 50% mniejszy rozmiar niż JPEG przy tej samej jakości wizualnej
  • Do 20% mniejszy rozmiar niż WebP
  • Obsługa przezroczystości (kanał alfa) — jak PNG, ale znacznie lżejszy
  • Obsługa HDR i szerokiej palety kolorów (10-bit, 12-bit)
  • Kompresja stratna i bezstratna w jednym formacie

Jak używać AVIF na stronie?

Najlepsza praktyka to użycie elementu <picture> z wieloma źródłami:

Przykład kodu HTML

HTML 
<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Opis"
       width="800" height="600"
       loading="lazy">
</picture>

Przeglądarka automatycznie wybierze najlepszy obsługiwany format — AVIF jeśli jest obsługiwany, potem WebP, a na końcu JPEG jako fallback.

Why Do Images Slow Down Your Website?

Images are the heaviest elements on most websites. The average page loads over 1 MB of graphics alone — that’s often 50-70% of the total page size. Every extra kilobyte means longer load times, worse Core Web Vitals scores, and lower Google rankings.

The LCP (Largest Contentful Paint) metric directly depends on the size of the main image on the page. Google recommends LCP under 2.5 seconds — with heavy JPEGs, that’s often impossible on slower connections.

AVIF — The Next Generation Image Format

AVIF (AV1 Image File Format) is a modern image format based on the AV1 video codec. Created by the Alliance for Open Media (Google, Mozilla, Apple, Microsoft), it offers:

  • Up to 50% smaller file size than JPEG at the same visual quality
  • Up to 20% smaller than WebP
  • Transparency support (alpha channel) — like PNG, but much lighter
  • HDR and wide color gamut support (10-bit, 12-bit)
  • Both lossy and lossless compression in one format

How to Use AVIF on Your Website?

The best practice is to use the <picture> element with multiple sources:

HTML Code Example

HTML 
<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Description"
       width="800" height="600"
       loading="lazy">
</picture>

The browser will automatically choose the best supported format — AVIF if supported, then WebP, and finally JPEG as a fallback.
